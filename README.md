Ag-Intel - Fine-Tuning LLMs for Precision AgronomyProject OverviewAg-Intel is a specialized Generative AI project designed to transform general-purpose Large Language Models (LLMs) into Precision Agronomy Assistants. While standard models often provide generic agricultural advice, Ag-Intel is fine-tuned to interpret complex soil and environmental data to generate localized, expert-verified decision support and professional reports.  The project focuses on closing the "reasoning gap" in high-stakes agriculture where specific chemical dosages, localized diagnostic confidence, and precise resource management are critical for preventing crop loss.  Technical ArchitectureBase Model: Meta’s Llama-3.1-8B-Instruct, a Transformer-based text-to-text model.  Specialized Datasets:Kisan Vaani’s Agriculture-QA: Expert-verified agricultural advice.  Agri-342K Corpus: Large-scale agricultural natural language processing data.  Fine-Tuning Methodology: QLoRA (Quantized Low-Rank Adaptation). This approach "freezes" the 4-bit quantized model weights and only trains a lightweight adapter layer, making it feasible to run on a single consumer-grade GPU.  Infrastructure: Developed in Google Colab utilizing a Tesla T4 GPU.  Key Features & ExperimentsAg-Intel is evaluated through five specific diagnostic and strategic scenarios:  Disease Diagnosis: Identifying specific causes (e.g., groundnut necrotic areas) and providing exact chemical dosages.  Resource Management: Interpreting pH and nitrogen sensor data to calculate specific fertilizer or lime requirements.  Strategic Planning: Recommending crop rotations (e.g., legumes to fix nitrogen) to restore soil health.  Risk Mitigation: Warning about specific dangers like 'phytotoxicity' when mixing chemicals.  Narrative Synthesis: Converting raw sensor numbers into structured professional risk mitigation reports.  Implementation DetailsThe project utilizes the Hugging Face ecosystem, including the transformers, peft, bitsandbytes, and accelerate libraries.  Optimization Techniques4-bit Quantization: The model is loaded in a compressed state using load_in_4bit, reducing the VRAM footprint from ~16GB to approximately 5.5GB to ensure stability on a T4 GPU.  PEFT/LoRA: Instead of updating all 8 billion parameters, only a small "adapter" layer is trained to inject domain-specific "ag-speak" and expert knowledge.  Evaluation FrameworkModel success is quantitatively measured using the AgriBench-13K benchmark, specifically comparing "Diagnostic Reasoning" and "Recommendation Accuracy" against the non-specialized base model.
